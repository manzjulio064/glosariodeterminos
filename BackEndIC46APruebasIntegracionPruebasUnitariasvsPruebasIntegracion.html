Pruebas Unitarias vs Pruebas de Integración

¿Por qué hacer tests de integración?

Una vez que todos los componentes o módulos funcionan correctamente de manera independiente, debemos verificar el correcto flujo de datos entre los módulos dependientes.


Sin embargo, aunque todos los módulos de la aplicación se hayan probado en las pruebas unitarias, aún pueden existir errores por diversas razones. A continuación, te mostramos algunos ejemplos que demuestran la importancia de realizar tests de integración.

  • Cada módulo suele estar diseñado por un desarrollador de software cuya lógica de programación puede diferir de los de otros módulos. Por lo tanto, estas pruebas se vuelven esenciales para determinar el funcionamiento de los módulos en conjunto.
  • Para comprobar si la interacción de los módulos con la base de datos es correcta o no.
  • Para verificar incompatibilidad entre los módulos que pueda generar errores.
  • Para testear la compatibilidad del hardware con el software.
  • Para evitar errores si el manejo de excepciones es inadecuado entre los módulos.
