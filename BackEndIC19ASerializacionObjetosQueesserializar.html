¿Qué es serializar?

Serializar un objeto

A partir de lo visto en el video, analicemos en profundidad a qué se refiere el concepto ‘‘serializar’’.

Para que un programa Java pueda convertir un objeto entero con todos sus atributos en un montón de bytes y pueda luego recuperarlo, el objeto necesita ser serializable. Al poder convertir el objeto a bytes, el mismo se puede, por ejemplo, enviar a través de la red o guardarlo en un archivo y después reconstruirlo al otro lado de la red o leerlo del archivo.

Para que un objeto sea serializable basta con que implemente la interfaz << Serializable >>. Como esta interfaz no tiene métodos, es muy sencillo implementarla. Basta con escribir implements Serializable y listo. Por ejemplo, la clase Persona es serializable y Java sabrá perfectamente enviar un objeto del tipo Persona por la red, escribirlo en un archivo o reconstruirlo a partir de un archivo. 

    
        import java.io.*;

        public class Persona implements Serializable {
            private String nombre;
            private String apellido;
            private int DNI;
        }
Si dentro de una clase que queremos serializar hay atributos que son otras clases,
estas, a su vez, también deben ser serializables. La clase ArrayList de Java, por
ejemplo, es serializable con lo cual si queremos guardar una lista con todos sus
elementos en un archivo, dichos elementos deben ser objetos de clases que imple-
menten serializable.

ObjectOutputStream y ObjectInputStream

Para poder convertir un objeto en un array de bytes, necesitamos usar la clase ObjectOutputStream y para poder reconstruir un objeto a partir de un array de bytes la clase ObjectInputStream. A continuación, veremos un ejemplo en Java de cómo guardar un objeto en un archivo y luego cómo recuperarlo. 

    
        import java.io.*;
