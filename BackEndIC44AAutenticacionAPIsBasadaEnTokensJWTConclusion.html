A continuación, te invitamos a mirar el live coding donde ejemplificamos lo visto anteriormente y te proponemos un desafío para validar lo aprendido.

¡Manos a la obra!

Vamos a utilizar una API para abstraernos de toda la complejidad de crear un token, la vamos a encontrar en el archivo template.zip. Tenemos que importarlo como un proyecto nuevo y arrancar desde ahí.

¿Qué debemos hacer?

  • Observar que existe una clase MyUserDetailsService que en el método loadUserByUsername devuelve un usuario. Modificarlo para que el username y el password sean “test”. Esta clase y método es el que se utiliza cuando se invoca a la API que genera el token JWT para validar el usuario y password.
  • Crear una clase HelloController y un endpoint GET (/hello) que solo devuelva un String “Hola Mundo JWT”.
  • A continuación, llamamos desde postman a la API /authenticate para generar un token, el body que debemos usar es {"username" : "test", "password" : "test"} y después debemos usar este token para invocar a la API /hello —todas las APIs excepto /authenticate necesitarán del token para poder funcionar—.
