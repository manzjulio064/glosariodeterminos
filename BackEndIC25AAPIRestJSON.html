JSON

Vimos que con Rest las respuestas a las peticiones se hacen siempre en el lenguaje de intercambio de información normalmente XML o JSON, que tanto el cliente como el servidor entienden.

Entonces, podemos decir que JSON es un formato de texto ligero para el intercambio de datos. Para las personas es simple de leer y escribir, mientras que para las máquinas también es fácil interpretarlo y generarlo.

Veamos un ejemplo:

    
        {
        "nombre" : "Homero",
        "apellido" : "Simpson",
        "hijos" : ["Bart", "Lisa", "Maggie"],
        "edad": 40,
        "esposa" : true
        }
