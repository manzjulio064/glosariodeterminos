Modelo DER

Seguramente recuerden de cuando cursaron Base de Datos que un sistema de gestión de base de datos relacional administra la información que deseamos guardar en diferentes tablas, como si fueran hojas de Excel.

Sin embargo, la diferencia entre una base de datos relacional y una planilla de Excel es que la base de datos administra relaciones entre tablas a través de claves.

Un registro en una tabla tiene una clave primaria que lo identifica unívocamente. Cuando una tabla almacena en una columna una clave primaria de otra tabla, este campo es conocido como clave foránea. Este hecho permite que surjan relaciones entre tablas. Es decir, el hecho de persistir una clave primaria de una tabla en una columna de otra tabla crea una relación entre estas.
