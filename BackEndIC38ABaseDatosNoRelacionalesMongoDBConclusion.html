Fixture web

Te proponemos el siguiente desafío para practicar los temas vistos.


Una casa de apuestas muy importante quiere agregar a su web una sección en donde se muestre el fixture de diferentes torneos y los resultados en tiempo real.


Para esta etapa vas a modelar los torneos.


Ya que el acceso a esta información será de forma masiva y recurrente, vas a utilizar una base de datos no relacional, en este caso MongoDB.

Luego del relevamiento obtuviste que:


Un torneo está compuesto por:

  • ID.
  • Nombre.
  • País.

¿Cuáles son los requerimientos?

  • Crear una base de datos local llamada “apuestas”.
  • Configurar la conexión en application.properties.
  • Agregar dependencias necesarias al proyecto.
  • Agregar las clases necesarias (Model, Controller, Service, Repository).
  • La API de los torneos deberá permitirte:
  • Agregar torneos.
  • Listar todos los torneos.
